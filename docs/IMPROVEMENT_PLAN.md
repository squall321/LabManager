# LabManager 개선 계획

UX 평가 + 코드 점검에서 도출한 개선 항목을 우선순위별로 정리하고, **하나씩 클리어**한다.
완료 시 `[x]` 체크 + 커밋 해시를 기록한다.

> 진행 규칙: 항목별로 구현 → 검증(빌드/E2E) → 커밋·푸시 → 이 문서 체크.

## 🔴 P0 — 보안·데이터 안전 (운영 전 필수)

- [ ] **P0-1 DB 마이그레이션 (Alembic)** — `create_all` 의존 제거. 스키마 변경 시 데이터 보존.
- [x] **P0-2 첫 로그인 인증 강화** — `SIGNUP_CODE` 도입. 설정 시 등록 이메일+가입 코드를 모두 알아야 계정 활성화(이메일 선점 차단). check-email이 `signup_required` 반환, 로그인 화면에 코드 입력. *(c… 보안 클러스터)*
- [x] **P0-3 SECRET_KEY 안전화** — 커밋된 기본키 제거, env 우선 + 미설정 시 `data/.secret_key`에 배포별 무작위 키 자동 생성(gitignore). `.env.example` 추가.
- [ ] **P0-4 테스트·CI** — 핵심 플로우 pytest + GitHub Actions(백엔드 E2E, 프론트 빌드).
