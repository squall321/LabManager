# LabManager 개선 계획

UX 평가 + 코드 점검에서 도출한 개선 항목을 우선순위별로 정리하고, **하나씩 클리어**한다.
완료 시 `[x]` 체크 + 커밋 해시를 기록한다.

> 진행 규칙: 항목별로 구현 → 검증(빌드/E2E) → 커밋·푸시 → 이 문서 체크.

---

## 🔴 P0 — 보안·데이터 안전 (운영 전 필수)

- [x] **P0-1 DB 마이그레이션 (Alembic)** — Alembic 도입(메타데이터 연동, SQLite batch 모드), 초기 마이그레이션(전체 10테이블) 생성·검증. 운영은 `alembic upgrade head`로 데이터 보존하며 스키마 변경. 개발은 기존 `create_all` 유지. README에 워크플로우 문서화.
- [x] **P0-2 첫 로그인 인증 강화** — `SIGNUP_CODE` 도입. 설정 시 등록 이메일+가입 코드를 모두 알아야 계정 활성화(이메일 선점 차단). check-email이 `signup_required` 반환, 로그인 화면에 코드 입력. *(c… 보안 클러스터)*
- [x] **P0-3 SECRET_KEY 안전화** — 커밋된 기본키 제거, env 우선 + 미설정 시 `data/.secret_key`에 배포별 무작위 키 자동 생성(gitignore). `.env.example` 추가.
- [x] **P0-4 테스트·CI** — `backend/tests/` pytest 12케이스(인증·가입코드, 불편함 격리·기본비공개, 미션·프롬프트·소유권, 공유/역추적 차단, 파트장 권한·N=5 익명, 템플릿 익명, 성장 회고→역량) + GitHub Actions(백엔드 pytest + 프론트 빌드).

## 🟠 P1 — 사용 경험

- [x] **P1-5 모바일 반응형** — 모바일 상단바+햄버거, 슬라이드인 사이드바 드로어(오버레이, 라우트 변경 시 자동 닫힘), `lg:ml-64` 반응형 본문 여백/패딩. 설문 sticky 푸터 패딩도 반응형 정렬.
- [x] **P1-6 칸반 터치 DnD** — 네이티브 HTML5 DnD를 `@dnd-kit/core`로 교체. Pointer/Touch 센서(활성 거리·지연 제약)로 마우스+터치 모두 지원, 드래그 핸들(그립)로 버튼/탭과 충돌 방지. dnd-kit은 보드 라우트 청크에만 포함(초기 로드 영향 없음).
- [x] **P1-7 코드 스플리팅** — 라우트 `lazy()` + Suspense, vendor 청크 분리(react/charts/motion). 920KB 단일 청크 → 라우트별 분할, recharts는 차트 페이지에서만 로드. 빌드 경고 해소.
- [x] **P1-8 Birkman 모델 고지** — 리포트 하단에 "자체 성향 모델 · 공식 Birkman과 다를 수 있음 · 평가/선발 근거 아님" 고지 추가.
- [x] **P1-9 통합 홈** — 대시보드에 WorkCraft 성장 스냅샷(불편함/미션/완성/배운점 + 역량 + CTA) 추가. 설문 문항 수 표기 stale(60→실제값) 수정.

## 🟡 P2 — 완성도·확장성

- [x] **P2-10 미션 상세 보기** — 보드 카드 제목 클릭 시 전체 내용 읽기 모달(+수정/명세서 진입).
- [x] **P2-11 템플릿→미션 생성** — 템플릿 카드의 "미션 만들기"가 제목·내용을 미션 빌더에 시드. 공유→재사용 순환 완성.
- [x] **P2-12 캘린더 개선** — 아젠다에 "일정 미지정" 진행 미션 목록 추가(클릭 시 날짜 지정 편집으로). *(기간 막대·드래그 리스케줄은 후순위)*
- [x] **P2-13 접근성(1차)** — 토스트 `aria-live`+닫기 라벨, 미션 상세 모달 `role=dialog`/`aria-modal`/ESC 닫기, 캘린더 이동 버튼 aria-label, 아이콘 버튼 접근 가능한 이름. *(전체 포커스 트랩/대비 점검은 후속)*
- [x] **P2-14 익명 집계 동의 경계** — 불편함 입력 화면에 고지 추가: 내용·이름은 비공개, 유형별 익명 집계는 같은 유형 5명 이상일 때만 파트장 지원 목적에 활용. (N=5 결정과 일관된 투명 고지 방식)
- [ ] **P2-15 페이지네이션** — 목록 전건 반환 개선.
- [ ] **P2-16 토큰 보관 강화** — localStorage→고려, 리프레시 흐름.
- [ ] **P2-17 운영 산출물** — 후순위(배포 계획 시 진행). *(완료: `.env.example`, `AUTO_CREATE_ALL` 운영 토글. Docker는 배포 시점에)*
- [x] **P2-18 C++ 모듈 상태 명시** — README에 "미빌드 참조 구현, 채점은 Python 경로" 명시.
